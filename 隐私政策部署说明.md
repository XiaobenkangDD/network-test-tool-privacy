# 网络测试工具 - 隐私政策部署说明

## 文件说明

- **`隐私政策.html`** - 完整的隐私政策网页文件

## 部署方式

### 方式一：GitHub Pages（免费，推荐）

1. **创建GitHub仓库**
   - 登录GitHub，创建新仓库（如：network-test-tool-privacy）
   - 设置为公开仓库

2. **上传文件**
   - 将 `隐私政策.html` 上传到仓库
   - 可以重命名为 `index.html` 或 `privacy.html`

3. **启用GitHub Pages**
   - 进入仓库 Settings → Pages
   - Source 选择 main 分支
   - 保存后等待几分钟

4. **获取URL**
   - 您的隐私政策URL将是：
     - `https://[您的用户名].github.io/network-test-tool-privacy/privacy.html`
     - 或 `https://[您的用户名].github.io/network-test-tool-privacy/`（如果使用index.html）

### 方式二：使用免费静态网站托管服务

#### Netlify（推荐）
1. 访问 https://www.netlify.com
2. 注册账号（可用GitHub账号登录）
3. 拖拽 `隐私政策.html` 文件到部署区域
4. 获得免费URL：`https://[随机名称].netlify.app/privacy.html`

#### Vercel
1. 访问 https://vercel.com
2. 注册账号
3. 上传HTML文件
4. 获得免费URL

#### Cloudflare Pages
1. 访问 https://pages.cloudflare.com
2. 注册账号
3. 上传文件
4. 获得免费URL

### 方式三：使用自己的网站

如果您已有网站：
1. 将 `隐私政策.html` 上传到您的网站服务器
2. 确保可以通过HTTPS访问
3. 记录完整的URL地址

## App Store填写说明

在App Store Connect中填写隐私政策URL时：

1. **进入应用信息**
   - 登录 App Store Connect
   - 选择您的应用
   - 进入"App隐私"或"应用信息"页面

2. **填写隐私政策URL**
   - 找到"隐私政策网址"字段
   - 输入完整的HTTPS URL
   - 例如：`https://yourusername.github.io/network-test-tool-privacy/privacy.html`

3. **验证URL**
   - 确保URL可以通过浏览器正常访问
   - 确保使用HTTPS协议（App Store要求）
   - 确保页面内容完整显示

## 注意事项

1. **必须使用HTTPS**：App Store要求隐私政策URL必须使用HTTPS协议
2. **必须可访问**：确保URL在审核期间可以正常访问
3. **内容完整**：确保隐私政策内容完整，符合App Store要求
4. **更新日期**：记得更新"最后更新日期"
5. **联系信息**：在隐私政策中填写真实的联系方式

## 常见问题

**Q: 必须要有自己的网站吗？**
A: 不需要。可以使用GitHub Pages、Netlify等免费服务。

**Q: URL必须是HTTPS吗？**
A: 是的，App Store要求必须使用HTTPS。

**Q: 隐私政策可以修改吗？**
A: 可以，但修改后需要更新"最后更新日期"。

**Q: 审核期间URL必须可访问吗？**
A: 是的，审核人员会访问您的隐私政策URL。
